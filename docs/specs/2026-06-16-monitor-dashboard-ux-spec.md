# 监控台 UX 优化：动态当前任务标题 + 跨项目总览/聚焦 + 项目切换交互 Spec

- 日期: 2026-06-16
- 状态: 待确认
- 范围: 移动端监控台（Dashboard）为主，配套 `local-agent` 后端元数据与跨项目聚合接口；桌面端复用同字段但布局不改
- 关联: `2026-06-15-mobile-nav-dashboard-spec.md`（监控台与 `attention` 元数据的前置实现）

## 1. 背景与痛点

当前监控台已能按「需处理 / 运行中 / 最近完成」分组（见 `apps/web/components/Dashboard.tsx:250-265`），但有三个体感痛点：

1. **标题对不上"当前在做的事"（核心痛点）**
   `deriveTitle()`（`packages/local-agent/src/util/claude-jsonl.ts:251`）优先用 `ai-title`，否则取**第一条用户消息**截断 80 字；`preview` 也是第一条用户消息前 140 字。用户常在一个 session 内连续处理多个任务（做完一个接着下一个），标题/预览却永远停在"开局快照"，无法反映当前子任务。

2. **项目作用域语义不清**
   `switchProject`（store）切项目时只加载该项目的 sessions，监控台实际是"单项目作用域"，但 UI 无强提示，"快速切换项目"被放在最底部，体感上分不清"在看谁"。用户期望：**默认看全部项目总览，选中某项目才进入聚焦过滤。**

3. **卡片信息密度低**
   分类有了，但卡片正文仍是静态 title，看不出"具体在等我批准什么 / 问我什么 / 刚跑完什么"。

## 2. 目标

1. 监控台卡片**动态反映会话当前任务/动作**，而非开局首句。
2. 监控台**默认跨项目总览**；选择/切换项目 = 进入单项目**聚焦过滤**，可随时返回总览。
3. **项目切换交互**优化：顶部常驻 pill 栏 + 活动徽章 + 可搜索下拉 + 侧滑手势，三者结合。
4. 卡片信息密度提升：需处理显示具体诉求、完成显示结果摘要。

非目标见 §8。

## 3. 核心痛点方案：动态当前任务（三层）

采用 **A + B + C 三层**，下层先顶住、上层逐步精准，**绝不留白**。

### 3.1 层 A — 最近用户指令（确定性兜底，零 LLM 成本）

- 新增 `lastUserText`：解析 JSONL 取**最后一条**用户消息文本（截断同 `deriveTitle` 规则），与现有 `firstUserText` 并存。
- 卡片标题优先级：`currentTask`(层 C) → `lastUserText`(层 A) → 现有 `title`。
- 这一层即可解决"接着处理下一个"场景的 90% 标题错位，纯解析、即时、可测试。

### 3.2 层 B — 运行时活动行（实时，按状态动态切换）

卡片副标题（活动行）随会话状态动态呈现，复用现有 WS delta（`server:claude_delta`）与 `attention` 字段：

- **运行中**（`driving`）：显示当前动作，从最新 `tool_use` 推导，如
  `正在编辑 Dashboard.tsx` / `运行 npm test` / `读取 12 个文件` / `思考中…`。
- **需处理**：
  - `question`：显示**具体问题文案**（AskUserQuestion 的问题首行）。
  - `approval`：显示**待批准工具名 + 关键参数**（如 `批准: Bash rm -rf dist`）。
  - `error`：显示错误首行。
- **已完成**（`done`）：显示**最后一条 assistant 消息首行**作为结果摘要。

活动行由前端从已有数据派生为主；`tool_use` 友好文案映射放在前端工具层（与现有工具渲染共用）。

### 3.3 层 C — Haiku 旁路滚动摘要（最高准确度、零对话污染）

**关键约束：Haiku 是旁路观察者，只读不写，永不进入被观察会话的上下文。**

- **链路独立**：`local-agent` 后端发起**独立的** Haiku API 调用（与驱动该会话的 Claude 进程是两条链路），system 约束为"会话观察员，只输出一句 ≤20 字的当前任务描述，中文"。
- **只读 JSONL**：输入为目标会话 JSONL 的**增量片段**（自上次摘要以来的新事件）；**不 append 任何行**到 `session.jsonl`，不调用工具，不影响该会话的 token 计费与上下文窗口。
- **结果存旁路**：输出写入 sidecar 元数据（`<session-id>.meta.json` 或 store 内存），作为 `ClaudeSession.currentTask` 字段，复用现有 `server:claude_session_updated` WS 推前端。
- **触发节奏（已确认）**：**回合结束时触发** —— 监听 `claude:drive_done`，对增量做一次摘要；**一轮最多一次 Haiku 调用**。非 live 会话不摘要。Haiku 出结果前由层 A 顶住。
- **成本控制**：增量为空则跳过；摘要与上次相同则不推送；可经 env 开关整体关闭（默认开）。
- **模型**：`claude-haiku-4-5`（最新 Haiku）。

> 比喻：Haiku 像旁边的记录员在白板上写"现在在做 X"，从不插嘴、不递纸条进你的对话。

## 4. 项目总览 / 聚焦方案

### 4.1 默认跨项目总览

- 监控台默认聚合**所有** `~/.claude/projects/*` 的会话，按「需处理 / 运行中 / 最近完成」分组。
- 后端新增**跨项目聚合**能力：在现有"单项目"基础上增加"全部项目"的会话列表来源（避免逐项目串行拉取的延迟）。
- 卡片需显示**所属项目名**，以便总览态区分来源。

### 4.2 聚焦过滤

- 顶部 pill 栏选中某项目 → 进入**聚焦**：三组仅显示该项目会话，顶部高亮当前项目。
- 常驻"全部"pill，一键回总览。
- 聚焦状态在前端为派生过滤（基于已聚合的会话集按 `cwd`/项目过滤），不必每次切换都打后端，降低延迟；URL 以 `?p=` 同步，`?p=` 缺省即总览。

## 5. 项目切换交互（三者结合）

1. **顶部 sticky pill 栏**：横向滚动，首位为"全部"，其后按活动度排序的项目 chip。每个 chip 带**活动徽章**：`运行 N` / `待处理 M`（M>0 时用珊瑚色强提示），一眼看出哪个项目需要你。
2. **可搜索下拉**：项目数多时，pill 栏末尾"更多"展开可搜索下拉（复用现 `ProjectPicker` 思路）。
3. **侧滑手势**：监控台主体区左右滑切换上一个/下一个项目（含"全部"），与 §4.2 聚焦联动。

## 6. 数据结构与影响（预估）

### 6.1 `@mac/shared` schema（`packages/shared/src/schemas.ts`）
`ClaudeSessionSchema` 新增可选字段（向后兼容，旧前端忽略）：
```
lastUserText: z.string().optional()   // 层 A：最后一条用户消息文本
currentTask:  z.string().optional()   // 层 C：Haiku 旁路摘要
```
（`projectName` 若现有 `ClaudeSession` 未直接携带，则在聚合接口补上，便于总览态展示。）

### 6.2 `local-agent`
- `util/claude-jsonl.ts`：新增 `lastUserText` 派生（与 `firstUserText` 同处计算）。
- `claude-store.ts`：
  - 在会话元数据中带出 `lastUserText`。
  - 跨项目聚合会话列表来源（"全部项目"）。
  - 新增 Haiku 观察员模块：监听 `claude:drive_done` → 取增量 → 调 Haiku → 写 `currentTask` → 经 `server:claude_session_updated` 推送；env 开关与节流。

### 6.3 `apps/web`
- `components/Dashboard.tsx`：
  - 标题优先级 `currentTask → lastUserText → title`。
  - 活动行（层 B）按状态派生。
  - 顶部 sticky pill 栏 + 徽章 + 搜索 + 侧滑；总览/聚焦切换。
  - 卡片显示项目名。
- `store`（`apps/web/lib/store.ts`）：跨项目总览状态、聚焦过滤派生、`?p=` 缺省=总览。

### 6.4 配置
- 根 `.env` / `.env.example` / README：Haiku 摘要开关（如 `CURRENT_TASK_SUMMARY=on`）与模型名。

## 7. 验收标准

1. 一个 session 内连续切换任务后，监控台卡片标题反映**当前**任务（层 A 立即生效；层 C 在回合结束后给出更精准描述）。
2. Haiku 摘要开启时，被观察会话的 JSONL **无新增行**、token 计费与上下文不受影响（验证：摘要前后 `session.jsonl` 行数仅由真实对话增长）。
3. 运行中会话副标题显示当前工具动作；需处理显示具体问题/待批准工具；完成显示结果摘要。
4. 监控台默认显示**全部项目**会话并标注项目名；选中项目进入聚焦、"全部"可返回。
5. 顶部 pill 栏带 `运行N/待处理M` 徽章；可搜索；可侧滑切换。
6. Haiku 摘要一轮最多调用一次；env 关闭后回退到层 A/B，功能正常。
7. 桌面端外观/交互不回归。
8. `pnpm test` / `pnpm typecheck` 全绿（含 `lastUserText` 解析、聚合、schema 测试）。

## 8. 非目标

- 不改桌面端主从布局。
- 不引入前端路由库（继续 `?p=&s=` + 派生态）。
- 不做用量趋势/统计页。
- 不在层 B 引入额外 LLM（纯派生）。
- 不做删除会话。
- 不改 ASR 链路。

## 9. 已确认决策

- 核心痛点采用 **A+B+C 三层**，Haiku 为旁路观察者（只读、不污染对话）。
- Haiku 触发：**回合结束（`claude:drive_done`）**，一轮最多一次。
- 项目作用域：**默认全部项目总览，选择/切换才聚焦**。
- 切换交互：**sticky pill+徽章 / 可搜索下拉 / 侧滑手势 三者结合**。
- 全部内容写入**一份** spec，内部分层实施。
