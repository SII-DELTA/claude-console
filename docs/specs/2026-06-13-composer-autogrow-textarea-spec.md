# Spec — 输入框多行自适应高度（类微信）

日期: 2026-06-13

## 背景

`Composer.tsx` 的输入 textarea 当前 `rows={1}` 固定单行高度，内容超出时仅内部滚动，不会随输入增高。用户希望像微信：默认单行，多行输入时输入框自适应变高，到上限后再内部滚动。

## 目标

- 默认高度保持现状（键盘模式 `minHeight: 2rem`，语音模式转写框 `minHeight: 1.5rem`）。
- 输入换行/内容增多时，textarea 高度随内容自适应增长。
- 增长到上限（沿用现有 `max-h-40` / `max-h-32`）后，停止增高并内部滚动。
- 发送 / 清空后高度回落到默认单行。
- 外部 `prefill` 填入文本后高度也应同步适配。
- 键盘模式与语音模式两处 textarea 行为一致（共用同一 `textareaRef`）。

## 设计要点

- 经典 auto-resize：每当文本变化，先将 `el.style.height = "auto"` 再设为 `el.scrollHeight`，由 CSS `max-height` 作为上限、超出滚动。
- 用一个依赖 `text` 的 effect 统一驱动；`prefill` 改文本后由该 effect 自动触发，无需单独处理。

## 非目标

- 不改最大高度上限值、不改字号 / 内边距。
- 不引入第三方 auto-resize 库。

## 验证

- 多行输入时输入框平滑增高；超过上限后内部滚动且不再增高。
- 发送 / 清空后回到单行默认高度。
- 「填入输入框」后高度与内容匹配。
- typecheck + 现有测试通过。
